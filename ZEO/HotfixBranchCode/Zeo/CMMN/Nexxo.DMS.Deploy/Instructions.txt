System Requirements:
------------------------------
.NET Framework: 4.0 and above
.VisualStudio : VisualStudio 2010 and above 
.IIS 	      : IIS6 and above
.JDK		  : JDK 1.6 or above

LOcal Build and Deploy Instruction:
=====================================
Setup Instruction::
------------------------------

DMS Deploy Setup :
1.Update DMS.Config in Nexxo.DMS.Deploy with your local Sql Server credentials and Sancus Repository path for automation
2.One time Setup , Environment Variable:
   -Run powershell as Administratorand run following command
   	[environment]::GetEnvironmentVariable("EnableNuGetPackageRestore","Machine")
   -If it returns true then run 
	exit
   -else run
        [Environment]::SetEnvironmentVariable("EnableNuGetPackageRestore", "true", "Machine") 

Automation Setup:
1. Update sancus\slingshot\configuration\Application_config.properties As per Automation Manual
2. Update sancus\slingshot\slingshot.properties As per Automation Manual

Run Script Instructions:
------------------------------
1. Run powershell as Administrator 
2. Change directory to Nexxo.DMS.Deploy directory located in nxo-prod (ie. nxo-prod\CMMN\Nexxo.DMS.Deploy)
	CD <Nexxo.DMS.Deploy Folder path>
3.Run Following command 
	DMSDeploy.ps1-> Deploys DB,Monitor packages,SVC and Web
		Command : powershell -ExecutionPolicy ByPass -File .\DMSDeploy.ps1 
	
	DBDeploy.ps1 -> Deploys DB and Monitor packages
		Command : powershell -ExecutionPolicy ByPass -File .\DBDeploy.ps1 
	
	WebDeploy.ps1 -> Deploys SVC and Web
		Command : powershell -ExecutionPolicy ByPass -File .\WebDeploy.ps1

	SystemTest.ps1 -> Run Automation on hosted application (i.e Start Hub, Start Node and run gradlew )
		Command : powershell -ExecutionPolicy ByPass -File .\SystemTest.ps1
		
	DMSCI.ps1 -> Build and deploy DMS and start automation (i.e internally it calls DMSDeploy.ps1 and SystemTest.ps1)
		Command : powershell -ExecutionPolicy ByPass -File .\DMSCI.ps1
